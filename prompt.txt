DATA SOURCE
Connect directly to NIS_Exchange_Rates.csv (no published datasource).

Fields: Date, USD, GBP, JPY_100, EUR, CHF, updated_at

PARAMETERS & FILTERS
- Currency Selector: USD, GBP, JPY_100, EUR, CHF (default: USD)
- Date Range: Default = last 12 months (Start Date, End Date parameters)
- Time Granularity: Day / Week / Month / Quarter / Year (default: Day)

Parameter XML attributes:
- All parameters: role='measure'
- Date parameters (Start Date, End Date): type='quantitative'
- String/list parameters (Currency Selector, Time Granularity): type='nominal'

DYNAMIC DATE HANDLING

[Selected Date (Dynamic)]
Role:   Discrete Dimension
Type:   Calculated Field (string)

CASE [Parameters].[Parameter 2]

WHEN "Day" THEN
    STR(YEAR([Date])) + "-" +
    RIGHT("0" + STR(MONTH([Date])),2) + "-" +
    RIGHT("0" + STR(DAY([Date])),2)

WHEN "Week" THEN
    STR(YEAR([Date])) + "-W" +
    RIGHT("0" + STR(DATEPART('week',[Date])),2)

WHEN "Month" THEN
    STR(YEAR([Date])) + "-" +
    RIGHT("0" + STR(MONTH([Date])),2)

WHEN "Quarter" THEN
    STR(YEAR([Date])) + "-Q" +
    STR(DATEPART('quarter',[Date]))

WHEN "Year" THEN
    STR(YEAR([Date]))

END

RULES:
- datatype = string, role = dimension, type = nominal
- Column-instance suffix: :nk (nominal), not :ok (ordinal)
- Use [Selected Date (Dynamic)] as X-axis in ALL time-series charts
- Formats are year-first (YYYY-MM etc.) so they sort correctly alphabetically
- Formula must reference [Parameters].[Parameter 2], NOT [Time Granularity]

───────────────────────────────────────────────────

[Period Label]
Role:   Discrete Dimension
Type:   Calculated Field (string, alternate format)

CASE [Parameters].[Parameter 2]
WHEN "Year"    THEN STR(YEAR([Date]))
WHEN "Quarter" THEN "Q" + STR(DATEPART('quarter',[Date])) + "-" + STR(YEAR([Date]))
WHEN "Month"   THEN RIGHT("0" + STR(MONTH([Date])), 2) + "-" + STR(YEAR([Date]))
WHEN "Week"    THEN "W" + RIGHT("0" + STR(DATEPART('week',[Date])), 2) + "-" + STR(YEAR([Date]))
WHEN "Day"     THEN STR(MONTH([Date])) + "/" + STR(DAY([Date])) + "/" + STR(YEAR([Date]))
END

NOTE: Period Label uses MM-YYYY display order (human-friendly but does not sort
chronologically across years). Use Selected Date (Dynamic) for chart axes;
Period Label for tooltips or manual override in Tableau Desktop.

───────────────────────────────────────────────────

[Period Change %]
Type: Table calculation (period-over-period)
Formula:
ZN((LOOKUP(AVG([Selected Rate]), 0) - LOOKUP(AVG([Selected Rate]), -1))
   / ABS(LOOKUP(AVG([Selected Rate]), -1)))
default-format: p1%
Used by: Period Change Bars (Sheet 9)

[Cumulative % Change]
Internal name: [Calculation_0697156532551683]
Type: Table calculation (cumulative from first period)
Formula:
ZN(
    (AVG([Selected Rate]) - LOOKUP(AVG([Selected Rate]), FIRST()))
    / ABS(LOOKUP(AVG([Selected Rate]), FIRST()))
)
Used by: Volatility Line (Sheet 8)
NOTE: This is NOT the same as Period Change %. Volatility Line shows
cumulative drift from the starting period, not period-over-period change.

KNOWN PROGRAMMATIC NOTES
- KPI customized-label CAN be set in TWB XML — use <customized-label> with
  three <run> elements for the field reference: "<", field_ref, ">", then "₪"
- Dual-axis on Main Trend Line: put same measure twice on rows shelf as (A + A)
- Multi-Currency rows shelf uses [Multiple Values], not [avg:Measure Values:qk]
- user:op='manual' on Measure Names filter goes on UNION groupfilter, not children
- Default Time Granularity: "Day" (users switch to Month/Quarter as needed)
- All parameter role='measure' (not 'dimension')
- Date params: type='quantitative'; list params: type='nominal'
